Overview:
A Java/JavaFX application containing four multiplayer games, each game requiring at least two players, where they will
compete against each other, through a graphical user interface.

My Role:
  - Implemented a Connect 4 game using the MVC architecture pattern
  - Utilized the Observer design pattern to update the view in response to player moves.
  - Developed algorithms to evaluate all winning conditions for connecting four pieces.
  - Created a feature which allows the players to save a game that has been started but not finished using serializable.
  - Wrote Javadoc documentation for all Connect 4 components along with this application's main menu files.
  - Assisted in creating the UML diagram for the multiplayer games application.
